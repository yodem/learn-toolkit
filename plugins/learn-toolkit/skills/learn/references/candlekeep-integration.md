# CandleKeep Integration Reference

## Overview

CandleKeep provides bidirectional integration with the learn workflow:
- **Read** (Phase 0.5): Scan library for existing knowledge on the topic
- **Write** (Phase 5.5): Upload compiled research book back to the library

CandleKeep is **never required** — the workflow proceeds normally if `ck` is not installed.

## CLI Detection (Phase 0)

```bash
ck --version 2>/dev/null && echo "CK_CLI=true" || echo "CK_CLI=false"
```

Set `HAS_CANDLEKEEP = true/false`. Missing = not an error, just skip silently.

## Library Scan (Phase 0.5)

**Condition:** `HAS_CANDLEKEEP = true` AND no `--no-ck-read` flag.

### Step 1: List items and match by topic

```bash
ck items list --json
```

Parse the JSON output. Match items whose title or description contains keywords from the topic. Select up to 3 most relevant items.

### Step 2: Read matching items

For each matching item (max 3):

```bash
ck items toc <item-id>
```

Review the table of contents, identify relevant pages/sections.

```bash
ck items read "id:<relevant-pages>"
```

Extract content snippets (500-1000 words per item).

### Step 3: Store results

Store as `ck_sources[]`:
```json
[
  {
    "id": "item-id",
    "title": "Document Title",
    "content_snippet": "Extracted relevant content..."
  }
]
```

Report to user:
- Found matches: `"Found X relevant documents in CandleKeep: [titles]"`
- No matches: `"No relevant library documents found"`

## Source Hierarchy (Phase 2)

CandleKeep sources rank between official docs and tutorials:

```
official docs > library (CandleKeep) > tutorials > blog posts > code repos
```

List CandleKeep sources under a "Library Sources" heading in the research summary.

## NotebookLM Loading (Phase 3)

Add CandleKeep content as text sources to NotebookLM **before** URLs, after notebook creation:

```
mcp__notebooklm-mcp__source_add(
  notebook_id=<id>,
  source_type="text",
  title="Library: [Item Title]",
  text=<content_snippet>,
  wait=false
)
```

Max 3 items = negligible impact on the 50-source limit.

## Local MD Files (Phase 2.5)

Always save research to `/tmp/learn-<topic-slug>/`:

```
/tmp/learn-<topic-slug>/
  README.md              — index with TOC, metadata, date
  research-summary.md    — 500-word synthesis
  sources/
    01-official-docs.md
    02-library.md         — CandleKeep sources (if any)
    03-tutorials.md
    04-articles.md
```

Each source file contains:
- URL or source identifier
- Title
- Backend that provided it (Tavily, Exa, CandleKeep)
- Content snippet

The `topic-slug` is the topic lowercased, spaces replaced with hyphens, special chars removed. Example: `"Kafka Event Streaming"` → `kafka-event-streaming`.

## Book Compilation (Phase 5.5)

**Condition:** `HAS_CANDLEKEEP = true` AND `--ck-write` flag present.

### Step 1: Compile book

Write to `/tmp/learn-<topic-slug>/book.md`:

```markdown
# [Topic] - Research Compendium

> Auto-generated by learn-toolkit on [YYYY-MM-DD]

## Executive Summary

[500-word synthesis from Phase 2 research summary]

## Chapter 1: Core Concepts

[Fundamental concepts, definitions, key terminology]

## Chapter 2: Architecture & Design

[System design, architecture patterns, component relationships]

## Chapter 3: Implementation Guide

[Practical how-to, code examples, configuration]

## Chapter 4: Best Practices & Patterns

[Recommended approaches, common pitfalls, optimization]

## Chapter 5: Ecosystem & Tools

[Related technologies, libraries, community resources]

## Appendix: Source Index

| # | Title | Source | Backend |
|---|-------|--------|---------|
```

Adapt chapter topics to match the actual research content — the template above is a starting point, not rigid.

### Step 2: Create CandleKeep item

```bash
ck items create "[Topic] - Research Compendium" --description "Auto-generated research compendium on [Topic]. Created by learn-toolkit on [date]." --no-session
```

### Step 3: Upload content

```bash
ck items put <item-id> --file /tmp/learn-<topic-slug>/book.md --no-session
```

### Step 4: Report

```
"Research book uploaded to CandleKeep (item #ID)"
```

## Error Recovery

See the Error Recovery table in `SKILL.md` for all CandleKeep error handling. The key principle: every CandleKeep failure is non-fatal — warn and continue.
